基础地址：http(s)://{host}:{port}
统一前缀：REST 接口均为 /api/**
统一返回体：Result<T>
{ "code": 0, "message": "OK", "data": {} }
日期时间格式：yyyy-MM-dd HH:mm:ss（见 JacksonConfig）
0. 通用约定
0.1 Result 结构
成功：code=0，message="OK"
失败：code!=0，data=null
参数错误：40000
未登录/登录过期：40100
无权限：40300
业务错误：40001
系统错误：50000
失败示例（401）：
{ "code": 40100, "message": "未登录或登录已过期", "data": null }

0.2 鉴权（JWT）
需要登录的接口：除白名单外均需要
Header：
Authorization: Bearer {token}
白名单（无需登录）：
/api/auth/**
/api/user/admin-config
/api/assets/background-presets
/api/news/**
/api/project/**
/api/music/**
/ws/**
/uploads/**、/avatars/**、/backgrounds/**、/background-presets/**
/news-images/**、/cover-images/**、/project-files/**、/music/**
/static/**、/、/index.html 等
1. 认证模块 Auth
1.1 登录
路由：POST /api/auth/login
是否鉴权：否
Content-Type：application/json
请求体：LoginRequestDto
{
  "username": "chen_ziyao",
  "password": "123456"
}
说明：username 字段兼容用户名或已绑定邮箱。

成功返回：Result<LoginResponseVo>
{
  "code": 0,
  "message": "OK",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiJ9....",
    "user": {
      "id": 1,
      "username": "chen_ziyao",
      "nickname": "陈子尧",
      "avatar": "https://example.com/avatar/admin.png",
      "role": "ADMIN"
    }
  }
}

失败返回（账号或密码错误）：40100
{ "code": 40100, "message": "账号或密码错误", "data": null }

1.2 发送邮箱验证码登录验证码
路由：POST /api/auth/login/email/code
是否鉴权：否
Content-Type：application/json
请求体：
{
  "email": "name@example.com"
}
成功返回：Result<Void>
说明：发送前先查询邮箱是否已经注册；未注册时返回“该邮箱尚未注册”，不会发送邮件。
查询保护：同一 IP 至少间隔 3 秒，每 10 分钟最多查询 20 次。
发送保护：同一邮箱 60 秒内只允许发送一次，同一 IP 每 10 分钟最多发送 10 次。

1.3 使用邮箱验证码登录
路由：POST /api/auth/login/email
是否鉴权：否
Content-Type：application/json
请求体：
{
  "email": "name@example.com",
  "code": "123456"
}
成功返回：Result<LoginResponseVo>

1.4 发送注册验证码
路由：POST /api/auth/register/code
是否鉴权：否
Content-Type：application/json
请求体：
{
  "email": "name@example.com"
}
成功返回：Result<Void>

1.5 注册
路由：POST /api/auth/register
是否鉴权：否
Content-Type：application/json
请求体：
{
  "username": "new_user",
  "password": "password",
  "nickname": "昵称",
  "email": "name@example.com",
  "code": "123456",
  "hr": false,
  "ledConfig": "{\"theme\":\"dark\",\"glow\":true,\"color\":\"#00e5ff\"}"
}
成功返回：Result<Void>
说明：前端会提交注册前正在使用的访客设置；未提交或内容无效时使用后端维护的默认设置。

6. WebSocket 实时聊天（公开连接）
6.1 连接地址
路由：WS /ws/chat?userId={用户ID}
示例：ws://localhost:8020/ws/chat?userId=1（开发环境经前端代理）
允许跨域：仅允许当前环境 app.cors.allowed-origins 配置的来源
注意：当前 WebSocket 连接鉴权是“简化版”，只用 userId query param 区分会话，没有校验 JWT。

6.2 客户端发送消息格式（Inbound）
消息模型：ChatInboundMessage
{
  "receiverId": "2",
  "content": "hello",
  "type": "TEXT",
  "fileUrl": null
}

字段：

receiverId：必填（用户 ID 或机器人 ID）
content：文本内容（TEXT 时使用）
type：TEXT / FILE / IMAGE
fileUrl：FILE/IMAGE 时可传
6.3 服务端下发消息格式（Outbound）
消息模型：ChatOutboundMessage
{
  "senderId": "1",
  "receiverId": "2",
  "type": "TEXT",
  "content": "hello",
  "fileUrl": null,
  "createTime": "2026-05-26 21:35:00"
}

行为说明（当前实现）：

如果 receiverId 对应的用户在线：服务端转发给对方 session
如果对方不在线：尝试把 receiverId 当作 sys_skill_bot.id，命中则走策略机器人回复，并回发给当前连接
聊天消息异步落库（用户发出、Bot 回复都会写 biz_chat_message）
Chatbot API Key 适配：
OpenAI 项目/服务账号 Key（sk-proj-、sk-svcacct-）自动请求 OpenAI Chat Completions 接口。
旧式 OpenAI Key 可在保存时添加 openai: 前缀；DeepSeek Key 可显式添加 deepseek: 前缀。
没有明确提供商前缀的普通 sk- Key 为兼容现有配置，默认按 DeepSeek 处理。
401、403、429 会分别返回面向用户的密钥无效、模型无权限、额度不足提示，不回显后端异常或密钥。
7. 数据库初始化
项目内不保留或自动执行 SQL 初始化脚本；现有 MySQL 数据库是唯一数据来源。
8. 常见错误响应
8.1 未登录（401）
{ "code": 40100, "message": "未登录或登录已过期", "data": null }
8.2 无权限（403）
{ "code": 40300, "message": "无权限访问", "data": null }
8.3 参数校验失败（40000）
{ "code": 40000, "message": "username: username不能为空; password: password不能为空", "data": null }
9. 当前已知差异/待你确认的点（避免前端对接踩坑）
WebSocket 连接目前不校验 JWT，仅靠 userId 区分会话（如果要上生产建议改成 token 鉴权）。

10. 音乐歌词（公开）
10.1 获取歌曲歌词
路由：GET /api/music/lyrics?trackId={歌单目录名/歌曲文件名}
是否鉴权：否
成功返回：UTF-8 text/plain
无歌词返回：HTTP 404
说明：切换歌曲时前端请求此接口。后端优先从配置的集中歌词目录读取 .lrc/.txt，
